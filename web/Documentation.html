SSH Alert Service

Your first line of defense against unauthorized server access. Get real-time Telegram alerts for SSH login attempts and enhance your system security.

About the Project

SSH Alert Service is a proactive security solution that keeps track of SSH login attempts on your server. It offers real-time notifications for both successful and failed login attempts, ensuring that you remain informed about who is trying to access your server and whether they are authorized to do so. With SSH Alert, you can monitor authentication logs and receive alerts directly to your Telegram account, providing a convenient and effective way to manage server security.

Key Features

  • Real-time Telegram notifications for SSH login attempts
  • Logging of all access attempts in alerts.log
  • Customizable alert configuration
  • Easy service management (start, stop, status)
  • Simple installation process

Installation

  1. Clone the SSHALERT repository:

    git clone https://github.com/rubikproxy/SSHALERT.git

  2. Navigate to the SSH_ALERT directory:

    cd /SSHALERT/

  3. Configuration:

    nano config.json

  4. Create a symbolic link for the systemd service:

    sudo ln -s /SSHALERT/sshalert.service /etc/systemd/system/sshalert.service

  5. Create a symbolic link for the init.d script:

    sudo ln -s /SSHALERT/sshalert /etc/init.d/sshalert

  6. Make the init.d script executable:

    sudo chmod +x /etc/init.d/sshalert

  7. Enable the service to start on boot:

    sudo systemctl start sshalert

  8. Start the sshalert service:

    sudo service sshalert start

  9. Verify the Status of sshalert service:

    sudo service sshalert status

After completing these steps, you will receive alerts on your Telegram whenever there is an SSH login attempt.
